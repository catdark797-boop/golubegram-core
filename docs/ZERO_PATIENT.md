# Protocol Zero Patient (Станция Инициализации)

## Вводная
Этот протокол применяется в условиях "чебурашнета" (полного отсутствия связи с внешним интернетом и корпоративными серверами). Устройства сотрудников физически не могут скачать приложение.

## Оборудование Станции Инициализации
- Ноутбук координатора Swarm Root Authority (MacBook)
- Кабели: USB-C -> USB-C, USB-C -> Lightning
- Установленное ПО: `Apple Configurator 2`, `Android Platform Tools (ADB)`
- Заранее скачанные (до блэкаута) артефакты:
  - `golubegram-android.apk`
  - `golubegram-ios.ipa`
  - `Swarm_Corporate_MDM.mobileconfig` (содержит Root CA)

## Процедура "Нулевой День"

### Для пользователей iOS (iPhone):
1. Сотрудник отдает свежий или сброшенный iPhone Координатору.
2. Координатор подключает iPhone кабелем к MacBook.
3. Открывает *Apple Configurator 2*.
4. Перетаскивает профиль `Swarm_Corporate_MDM.mobileconfig` на иконку устройства.
5. **UX Комикс-Гайд для сотрудника:** На экране смартфона появляется инструкция-комикс ("Бабушка-режим"), которая пошагово, большими картинками и стрелками показывает сотруднику:
   - "Нажми Настройки -> Основные -> VPN и управление устройством".
   - "Нажми Установить".
   - "Нажми Об этом устройстве -> Доверие сертификатам -> Включи тумблер для Swarm Root CA".
6. После активации профиля Координатор перетаскивает файл `golubegram-ios.ipa` на устройство через Apple Configurator.
7. Приложение установлено. Кабель отключается. Сотрудник уходит в P2P-рой.

### Для пользователей Android:
1. Сотрудник отдает устройство Координатору.
2. Координатор включает "Отладку по USB" (тап 7 раз по номеру сборки).
3. Подключает кабель.
4. В терминале выполняет: `adb install -r golubegram-android.apk`.
5. Корневые CA для Android вшиты напрямую в приложение (или устанавливаются отдельным скриптом через Intent-ы).
6. Кабель отключается. Устройство готово к работе в Mesh-сети.
